关于mvc annotation-driven和context annotation-config

发表于 2019-05-17 | 分类于 后端

<context:annotation-config />

作用是向Spring容器注册

AutowiredAnnotationBeanPostProcessor、CommonAnnotationBeanPostProcessor、PersistenceAnnotationBeanPostProcessor 以及 RequiredAnnotationBeanPostProcessor 这 4 BeanPostProcessor。

注册这四个BeanPostProcessor的作用是让程序能够识别相应的注解。

例如：

  • @Autowired
  • @ Resource 、@ PostConstruct、@ PreDestroy等
  • @PersistenceContext
  • @Required

但是<context:annotation-config />的扫描粒度是很大的，我们还可以使用<context:component-scan base-package=”XX.XX” /> 来扫描包路径，定义Bean，并把Bean注入容器中使用。

context:component-scan做了context:annotation-config要做的事情，还额外支持@Component，@Repository，@Service， @Controller @RestController, @ControllerAdvice, and @Configuration 注解。

所以如果使用了<context:component-scan base-package=”XX.XX” />来扫描包路径，就可以把<context:annotation-config /> 去掉了。

<mvc:annotation-driven />

添加注解驱动，使注解标注的类生效。用来匹配请求的controller

它在容器中注册了以下Bean：

  • RequestMappingHandlerMapping
  • BeanNameUrlHandlerMapping
  • RequestMappingHandlerAdapter
  • HttpRequestHandlerAdapter
  • SimpleControllerHandlerAdapter
  • ExceptionHandlerExceptionResolver
  • ResponseStatusExceptionResolver
  • DefaultHandlerExceptionResolver

那上面这些Bean是做什么的呢？

前二个是HandlerMapping接口的实现类，用来处理请求映射的。

  • 其中第一个处理@RequestMapping注解的。
  • 其中第二个会将controller类的名字映射为请求url。

中间三个是用来处理请求的，即确定用哪个Controller的哪个方法处理当前请求。

后三个是处理异常的。

另外支持使用@RequestBody和@ResponseBody注解。

ps:

  • 当使用mvc:view-controller标签时一定要加入mvc:annotation-driven，不然会使requestMapping失效。
  • 当为了处理静态资源问题而加入mvc:default-servlet-handler时，也一定要加入mvc:annotation-driven，不然requestMapping同样会失效。
  • 当使用自定义类型转换器的时候需要加上mvc:annotation-driven标签。
