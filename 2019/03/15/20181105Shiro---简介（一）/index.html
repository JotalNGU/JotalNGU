Shiro学习---简介（一）

发表于 2019-03-15 | 分类于 后端

概述

Shiro是一个Java平台的开源安全框架，它使用简单，功能强大，为开发人员提供了认证、授权、加密以及会话管理等功能。

与Spring Security对比

另一常用的安全框架是Spring内置的Spring Security，Spring Security是细颗粒化的，它功能比Shiro更加强大，但是使用起来更加复杂，可能在实际使用中不需要使用这么复杂的东西，Shiro就能够满足基本需求，所以在满足需求的情况下我们优先选择简单的Shiro。

从应用程序看Shiro架构

在Shiro中，有三个最重要的组件：Subject、SecurityManager、Realm。

  • Subject：代表当前“用户”，这个用户是抽象概念的，不一定是人，与当前应用交互的任何东西，都可以是Subject。
  • Security Manager：安全管理器，是Shiro的核心，Shiro中的所有安全操作都会与SecurityManager交互，负责进行认证、授权、会话以及缓存的管理，与SpringMVC中的DispatcherServlet扮演的角色相似。
  • Realm：域，Realm提供安全数据源给SecurityManager。

从Shiro内部来看Shiro架构

我们重点看一下Security Manager内部的组件。

  • Authenticator:认证器，负责主体的认证，例如用于用户登录的认证。
  • Authorizer：授权器，主要用于用户权限控制，即用户拥有什么权限能够访问应用中的哪些功能。
  • SessionDao：数据访问对象，用于会话的CRUD，比如我们想把Session保存到数据库，那么可以实现自己的SessionDAO，通过如JDBC写到数据库；比如想把Session放到Memcached中，可以实现自己的Memcached SessionDAO；另外SessionDAO中可以使用Cache进行缓存，以提高性能；
  • CacheManager：缓存控制器，来管理如用户、角色、权限等的缓存的；因为这些数据基本上很少去改变，放到缓存中后可以提高访问的性能。
  • Cryptography：密码模块，Shiro提高了一些常见的加密组件用于如密码加密/解密的。
